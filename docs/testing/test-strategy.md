# 测试策略

更新时间：2026-04-09

## 1. 当前测试现状

- 当前没有有效的业务级自动化测试
- 当前没有稳定的测试门禁流程
- 当前发布前验证仍以人工验证和基础构建验证为主

## 2. 当前最低验证要求

- `assembleDebug` 通过
- 桌面主页可以正常打开
- 应用管理页可以加载并保存勾选状态
- 电话联系人页可以加载、添加、编辑、删除和拨号
- 视频联系人页可以打开、添加和删除联系人

## 3. 当前人工测试清单

- 默认桌面切换后，主页入口正常
- 主页返回键提示正常
- 设置入口正常
- 天气入口在有对应厂商应用时可跳转
- 电话联系人权限申请正常
- 联系人新增、编辑、删除后页面刷新正常
- 视频联系人新增、删除后列表刷新正常

## 4. 后续自动化测试建议

- 单元测试
  - `ContactManager`
  - 排序和本地存储逻辑
  - 网络和权限判断的包装逻辑
- 仪器测试
  - 主页启动
  - 页面跳转
  - 联系人页基本交互
- 手工专项测试
  - 老年用户操作路径
  - 权限拒绝场景
  - 不同厂商系统兼容性

## 5. 发布门禁建议

- 第一阶段：构建通过 + 人工冒烟测试
- 第二阶段：补齐核心单元测试后，再引入自动化门禁
- 第三阶段：补齐 UI 冒烟测试和权限回归测试
