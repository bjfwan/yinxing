# 微信视频自动化规划

更新时间：2026-04-09
状态：规划中，当前暂停推进

## 1. 文档定位

本文档描述的是“未来是否继续做微信视频自动化”这一规划方向，不代表当前版本已经交付，也不代表当前仓库中的实验性代码可以直接作为产品能力上线。

## 2. 当前仓库真实状态

- 当前存在实验性的微信自动化代码
- 当前存在视频联系人页和微信视频入口
- 当前没有来电通知监听服务
- 当前没有微信视频自动接听完整链路
- 当前没有系统电话自动接听完整链路
- 当前没有稳定的自动化测试和兼容性基线

## 3. 当前为什么暂停

- 当前项目主目标仍是桌面、应用管理和电话主流程稳定化
- 微信自动化能力对兼容性、权限、设备状态和微信版本高度敏感
- 当前仓库的自动化实现还没有达到可承诺的稳定度

## 4. 如果未来恢复投入，前置条件应当包括

- 明确支持的微信版本范围
- 明确支持的手机品牌和系统版本范围
- 建立独立的兼容性测试矩阵
- 明确自动化失败后的用户兜底流程
- 在核心 Launcher 功能稳定后再进入开发排期

## 5. 未来恢复后的建议拆分

### Phase 1

- 稳定发起视频通话链路
- 完善状态检测、超时和失败回退

### Phase 2

- 补齐通知监听和来电识别
- 设计来电接听页面

### Phase 3

- 补齐自动接听配置
- 建立兼容性回归测试

## 6. 当前结论

- 当前代码库保留实验性自动化实现
- 当前产品文档不把它作为正式交付能力
- 当前优先级低于主桌面和电话模块治理
