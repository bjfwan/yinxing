# 当前功能状态

更新时间：2026-04-27

| 功能 | 状态 | 当前说明 |
| --- | --- | --- |
| 桌面主页 | 已实现 | 支持时间日期展示、顶部天气卡片、系统设置、电话、视频页和已选择应用入口；进入主页时整体淡入，分钟变化时时钟做轻微 fade pulse |
| 默认 Launcher 入口 | 已实现 | Manifest 已声明 `HOME` 和 `LAUNCHER` |
| 应用管理 | 已实现 | 支持勾选加入桌面和取消移除，基于 `LauncherAppRepository` 读取并缓存桌面应用 |
| 天气入口 | 已实现 | 主页顶部天气卡片可点击，优先尝试厂商天气应用，失败时回退到浏览器天气页；双接口并行加载 |
| 电话联系人 | 已实现 | 以应用内本地联系人为主，支持新增、编辑、删除、拨号、联系人照片与从系统通讯录导入；联系人读写全面迁移至后台线程；首屏前 8 项 stagger 淡入 |
| 视频联系人管理 | 已实现 | 页面可独立打开，支持本地新增、删除、搜索、置顶和按通话次数排序；首屏前 8 项 stagger 淡入 |
| 微信视频自动化拨号 | 已验证 | 在 vivo V2285A / Android 15 / 微信 8.0.66 上验证通过；无障碍服务采用 `SelectToSpeakService` 触发微信节点树开放 |
| 微信视频自动接听 | 未实现 | 没有通知监听服务和来电处理链路 |
| 系统电话自动接听 | 部分实现 | 已接入 `PHONE_STATE` 广播、来电页和 `TelecomManager.acceptRingingCall()` 主流程，兼容性与设备验证仍待补齐 |
| 自动化状态浮窗 | 已实现 | 服务于微信自动化链路的浮窗状态播报 |
| 自动化 TTS 播报 | 已实现 | 服务于微信自动化链路的语音播报 |
| 家属维护中心 | 已实现 | `SettingsActivity` 聚合电话/视频联系人、首页应用、自动接听、权限、设备和系统六类入口；联系人面板内嵌「桌面应用怎么整理」三步图文小贴士 |
| 暗色模式 | 已实现 | 跟随系统/常亮/常暗三档；切换时整页淡出再 recreate 后淡入，chip 颜色与 stroke 走 180ms 渐变 + 弹性点击反馈，不再硬切闪烁 |
| 低性能模式 | 已实现 | 关闭主页入场、时钟 pulse、联系人 stagger、暗色 chip 弹性等装饰性动画；列表 itemAnimator 同步关闭 |
| 自动化测试体系 | 已实现 | 113 个单元测试全绿（`testDebugUnitTest`），Robolectric UI 冒烟覆盖主干页面，设备级仪器测试脚手架已建立 |
| Baseline Profile | 已实现 | Profile 已嵌入包体，有 Profile 时 P90 帧时间相比无 Profile 降低约 46% |

## 说明

- "已实现"表示主流程在当前代码中已经存在
- "已验证"表示主流程在真实设备上跑通，但兼容性覆盖尚不完整
- "部分实现"表示存在入口，但仍有明显限制或兼容性缺口
- "未实现"表示需求或规划存在，但当前仓库没有完整交付链路
- 视频联系人页不再依赖实验性自动化权限才能打开，相关权限在真正发起微信视频时单独检查
- 微信自动化无障碍服务已合并为单一服务，用户在设置中只需开启一个无障碍项即可
