# OldLauncher 产品需求说明

更新时间：2026-04-28
状态：当前版本事实说明

## 1. 产品定位

- 类型：Android 本地桌面启动器
- 目标用户：需要更简单手机主页的老年用户
- 技术边界：单机运行，无后端服务，无账号体系
- 当前目标：把常用应用、电话联系人、视频联系人与基础系统入口收拢到一个更易操作的主页中

## 2. 当前版本目标

- 提供可作为默认桌面的主页
- 提供清晰的大图标入口和时间日期展示
- 允许家属或维护者选择桌面展示哪些应用
- 提供可直接拨号的电话联系人管理能力
- 提供可独立维护的视频联系人页
- 保留微信视频相关入口，但不把自动化能力作为当前版本承诺

## 3. 当前已纳入范围

### 3.1 桌面主页

- 支持 `HOME` 和 `LAUNCHER` 入口
- 桌面展示时间、日期和常用入口
- 内置入口包含：电话、微信视频、添加
- 顶部天气卡片展示天气信息，并作为天气入口使用
- 支持把已选择的第三方应用展示到桌面
- 在桌面按返回键时提示"已经是桌面了"

### 3.2 应用管理

- 提供应用列表页面
- 支持通过勾选把应用加入桌面
- 支持取消勾选后从桌面移除
- 当前使用本机 Launcher 应用列表进行选择

### 3.3 电话模块

- 以应用内本地电话联系人为主
- 支持从系统通讯录读取联系人并导入到本地联系人列表
- 展示联系人头像、姓名、号码
- 支持联系人拨号
- 支持新增联系人
- 支持编辑联系人
- 支持删除联系人
- 支持为联系人选择照片
- 当前新增、编辑、删除作用于应用内本地联系人，不回写系统通讯录
- 联系人所有读写操作在后台线程执行，不阻塞主线程

### 3.4 视频通话联系人页

- 提供独立的视频联系人页面
- 支持本地新增和删除视频联系人
- 支持搜索与置顶视频联系人
- 支持按通话次数排序联系人
- 保留发起微信视频的实验性入口

## 4. 当前未纳入交付范围

- 微信自动化拨号的稳定性承诺
- 微信视频来电自动接听
- 系统电话自动接听的全机型稳定性承诺
- 来电风险评估的全机型稳定性承诺（实验性功能）
- AI会员功能的完整交付（实验性功能）
- 通知监听服务
- 后台定时任务
- 远程协助、云同步、日志后台上报

## 5. 非功能要求

- 兼容 Android 7.0 及以上系统
- 本地存储优先，避免引入服务端依赖
- 页面风格以大字体、高对比、低学习成本为主
- 数据读写操作不阻塞主线程
- 目录和文档结构必须可维护，便于多人协作
- 基础验证至少覆盖单元测试、Debug 构建、Lint 与仪器测试编译

## 6. 当前验收口径

- `:app:testDebugUnitTest` 可以通过（当前 171 tests, 0 failed）
- `:app:assembleDebug` 可以通过
- `:app:assembleDebugAndroidTest` 可以通过
- `:app:lintDebug` 可以通过
- 主页、应用管理、电话模块和视频联系人页可以正常打开
- 主页作为默认桌面后，基础返回和入口跳转逻辑正常
- 文档内容与当前代码结构保持一致
