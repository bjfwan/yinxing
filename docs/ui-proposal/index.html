01

诊断 · 现在卡在哪里

先把问题摆出来，再谈方案。下面的每一条都对应代码里具体可定位的位置。

P0

夜间一片刺眼白

values-night/themes.xml 是空注释，bg_gradient.xml 写死浅色。爷爷半夜起来接电话直接亮瞎。

P0

拨号卡只有姓名没号码

老人无法肉眼确认"我要打的是不是这个号"，按错直接拨出，没有挽回机会。

P0

删除按钮和取消等宽并排

手抖一下就误删。危险操作必须收窄、降权、必要时长按确认。

P0

"长按删除应用"是隐藏手势

老年用户根本发现不了；同时又容易在桌面误触发。建议彻底关闭主页长按删除，统一走应用管理页。

P1

设置页绕过了 typography

activity_settings.xml 里 5 处直接写 textSize="22sp/17sp/15sp"，破坏主题统一，缩放无效。

P1

4 个页面 4 套 Hero padding

主页 12dp / 电话 24dp / 视频 24dp / 应用 20dp / 设置 16dp。应抽成 component_page_header.xml 公共组件。

P2

设置页 6 卡片瀑布流

每张占近半屏，要滚 2-3 屏才看完。建议按"使用频率 / 风险等级"分组。

P2

维护入口分散在 3 个地方

家属维护时要在主页、联系人页、设置页之间反复跳。建议长按桌面空白处 → 进入"维护中心"。

02

主页 · 时间 / 天气 / 应用网格

左：当前 activity_main.xml 渲染样貌。右：拟议改动。所有改动均不破坏现有 Adapter 与 RecyclerView 数据流。

为什么这么改

  1. 家属维护入口可达性：当前是右上角小字 TextButton，textColor 还是 muted，根本看不见。改为带状态指示的徽章按钮，并且统一为"家属维护"——含义比"家属设置"更准确。
  2. 电话/视频副标题：让爷爷一眼看到"里面有几个人"，降低进入空页面的概率，也让"添加联系人"成为隐式提示而非显式按钮。
  3. 移除"添加"卡片：把"添加应用"这种维护功能下沉到家属维护中心，桌面只留"老人会用的"。
  4. 关闭长按删除：避免误触；删除应用统一走应用管理页（已存在）。
  5. 农历用 pill 强调：从次要 muted 文字升级为带背景的小徽章，老人喜欢看农历日期。
03

电话联系人 · 把"号码"和"安全"还给老人

这是整个项目影响最大的一处改动。当前 220dp 高的卡片只有头像 + 姓名 + 一个按钮，没有显示号码。老人按错就直接拨出。

设计原则 · 给爷爷的桌面只增加信息，不增加操作

  1. 大头像保留：BEFORE 的 180dp 头像不缩小。头像支持本人真实照片，无照片时降级为名字首字。这是老人识别"按谁"的第一通道。
  2. 新增号码字段：mono 字体加粗 20sp，让爷爷拨打前能肉眼核对最后 4 位——这是当前最大的安全漏洞。
  3. 按钮缩小但仍足够大：56dp 高度（远超 48dp 的最小可点击规范），交互保持点击不变。不引入按住——长按和点击对老人区分不明显，会反复误触。
  4. "自动接听"小徽章：仅在该联系人启用时显示，11sp 不抢戏，但能让爷爷意识到"这个会自己接"。
  5. 移除"常拨"标签：靠位置就能传达常拨——家属把常用联系人排在第一位即可。少一个标签等于少一个干扰元素。
  6. 拨打点击区可配置：默认仅按钮可点（安全），家属可在《来电与接听》中切换为"整卡拨打"——看下面对比。

核心理念：设置是家属使用的，使用是老人使用的。所有面向老人的页面都要追求零新手势、零额外确认，但仅限于默认状态——不同老人习惯不同，重要的是让家属能针对性调整。

可配置 · 家属维护中心

04

设置 · 把 6 张卡片重新组织成 3 组

当前设置页是 6 张大卡的瀑布流，要滚动 2-3 屏才能看完。优化后按"状态优先 / 日常 / 高级"分组，前面紧凑后面折叠。

05

关键弹窗 · 仅在不可逆操作时弹出

原则：老人使用页面上的运行时弹窗越少越好。拨号、在家提示等干扰性弹窗一律取消。仅保留"删除联系人"这种不可逆操作的确认。

移除

拨号确认弹窗

拨号是老人高频操作，多一次确认等于多一次随机丟失。靠联系人卡片上的号码可见已经足够避免误拨。

移除

“在家”提示浮窗

老人在桌面按了返回？什么都不做就是最好的反馈。不在主页送 Toast 干扰。

移除

联系人保存成功 Toast

联系人页本身刷新后有可见变化，不需要额外反馈。减一个 Toast 少一个干扰。

保留

删除联系人确认

删除不可逆，这是唯一该有的防误触设计。并且只会出现在家属维护页，老人不会看到。

06

暗色模式 · 位置、入口、默认值

位置：家属维护中心 → 设备显示 → 深色模式。三选项：始终明亮 · 始终暗色 · 跟随系统。默认：跟随系统，顺应安卓全局设置。老人本人不需要接触这个开关。

设计决策：老人不应该亲自切暗色

  • 老人看不到入口：在主页顺手一点就切主题，会让老人误以为手机坏了。所以入口只在家属维护中心 → 设备显示。
  • 默认跟随系统：Android 11+ 多数手机夜间会自动全局切暗，爷爷不需交互。三选项只是给家属调整。
  • 不用纯黑：纯黑屏 + 亮色文字在老人眼中会产生镶边闪烁，选用暖深蓝灰 #0F1419 → #161B22可减轻。
  • 关键操作色不变：拨号绿、删除红、银杏黄保持色相。老人依赖颜色记忆识别按钮。
07

颜色与老年视觉 · 我们的配色不是凭感觉

你提了一个好问题：这套配色是否符合老人使用？下面是基于公开研究的逐条决策对应——每个颜色都不是"觉得好看"，而是"研究里有依据"。

01

晶状体黄变 (Lens Yellowing)

65 岁后晶状体颜色明显偏黄，对蓝/紫色波段透过率下降 25-50%；蓝紫色看起来偏黑、偏灰。

设计决策

不使用纯蓝/纯紫作为主语义色。"通讯/视频"图标用偏青的蓝而非纯蓝，并依赖形状 + 文字双通道传达，不依赖单一色相。

WHO World Report on Vision · 2019
02

对比敏感度下降 (Contrast Sensitivity)

75 岁人群对比敏感度仅为青年人的 1/3；WCAG AA 的 4.5:1 不够用，老人 UI 应达到 AAA 7:1。

设计决策

正文/页背景对比度 16:1（远超 AAA），按钮文字对比度 ≥7:1。所有"次要文字" (text_secondary, text_muted) 的对比度全部重测，不再低于 4.5:1。

WCAG 2.1 AAA · NN/g 2019
03

暗适应能力下降 (Dark Adaptation)

老人从亮到暗的眼睛适应时间是青年人的 2-3 倍；纯黑屏配亮色文字反而难以聚焦，会出现"镶边闪烁"。

设计决策

暗色模式不用纯黑 (#000)，改用暖深蓝灰 (#0F1419 → #161B22)，文字也不用纯白 (#FFF)，改用奶油色 (#F1F5F9)。

Owsley · "Aging and Vision" · 2011
04

暖色波段保留度 (Warm Spectrum)

视网膜锥细胞对红、橙、黄的识别能力衰退最慢；这三色一直到 80+ 仍保持高识别度。

设计决策

品牌色用银杏黄 (#B8882A)，危险色保持红 (#EF4444)，警示色用橙 (#F59E0B)。三色构成完整的"色彩警示系统"。

Werner · "Aging Eye and Color" · 2010
05

色盲共存率 (Color Blindness)

中国男性约 5-8% 有色觉缺陷，70+ 群体合并视力衰退后，红绿区分困难者更多。

设计决策

状态信息从不只用颜色传达：成功配 ✓ + "已启用"，警告配 ! + "需要注意"，错误配 ✕ + 文字。色盲老人也能完整识别。

WHO Global Disability Report
06

白底刺眼 (White Glare)

纯白背景 (#FFF) 在年长群体中常被报告"刺眼"，柔和暖白（米色/象牙色）显著降低疲劳感。

设计决策

明亮模式背景从默认 #F8FAFC 微调向暖色（保留作为兼容色），暗色模式背景偏暖深灰，避免冷调白屏。

Sloan · "Reading and Aging" · 2007

一句话总结

这套配色不依赖审美判断——每个色值都对应一条公开研究结论， 并且每个语义都双通道传达（颜色 + 形状 / 颜色 + 文字）， 色盲、白内障、黄斑变性老人也能完整使用。

PR 路线 · 4 个独立可上线的提交

每个 PR 都设计为不影响其他 PR 也能独立 ship，避免大爆炸式合并。

PR 1

  • 暗色 token 添加到 values-night/
  • Hero 头部抽 component_page_header.xml
  • 设置页 5 处硬编码字号回归 typography
  • 渐变背景启用 mid 色
风险 · 低 影响 · 全局 预计 · 2-3 小时
PR 2

联系人卡片重做

  • 卡片新增号码字段（item_phone_contact.xml / item_video_contact.xml）
  • 保持 180dp 头像 + 220dp 卡片高度不变
  • 拨打按钮缩小为 56dp，交互仍为点击（不引入按住）
  • “自动接听”徽章小化、移除“常拨”标签
  • 新增 enable_full_card_tap 偏好项（默认 false），电话与视频联系人共用
  • 删除联系人确认弹窗采用不对称布局
风险 · 低 影响 · 联系人页 预计 · 4-5 小时
PR 3

信息架构

  • 设置页拆为"状态 / 日常 / 高级" 3 组
  • "家属维护中心"重命名 + 入口收拢
  • 主页"添加"卡下沉
  • 关闭主页长按删除手势
风险 · 中 影响 · 设置页 + 主页 预计 · 3-4 小时
PR 4

交互细节

  • TTS 在大字模式下朗读卡片标题
  • 错误/加载态统一组件化
  • 首次使用引导（设默认桌面 / 家人 / 字号）
  • 错误拨号代以静默重试，不弹 Toast
风险 · 低 影响 · 体验细节 预计 · 4-5 小时

合计 12-16 小时 工作量，分 4 次提交，每次提交都有可演示的可用版本。 建议先做 PR 1 + PR 2，因为这两个对老人体验提升最直接。
