# 老人桌面 (OldLauncher) 需求文档
版本: v1.0
日期: 2026-02-18

## 1. 项目定位
- **类型**: Android Launcher (桌面启动器)
- **目标用户**: 老人
- **架构**: 纯本地应用，无后端服务器
- **核心目标**: 极简、易用、防误触，屏蔽老人不需要的复杂功能

## 2. 核心功能

### 2.1 桌面主页 (UI/UX)
- **布局模式**: **单页显示**，内容铺满一页，禁止左右或上下滑动，防止老人迷路。
- **预设内容**: 默认显示 **电话**、**微信**、**抖音** 三个核心应用。
- **视觉风格**: 极大图标、极大字体、高对比度。

### 2.2 功能入口
- **添加应用 (+)**:
  - 界面提供一个明显的“+”号按钮。
  - 点击后进入“应用管理列表”，列出手机内已安装的所有应用。
  - 操作方式：通过**勾选**的方式添加显示到桌面，取消勾选则移除。
- **系统设置**:
  - 提供一个明确的图标/入口，点击直接跳转至 Android **系统设置** 页面。
- **电话功能**:
  - 目前保留入口。
  - **待定**: 后续将开发特色的电话功能（如大头贴、快速拨号等），目前暂按需预留。

### 2.3 系统交互与权限
- **下拉通知栏**: **保留**，不屏蔽（允许老人查看电量、时间、通知）。
- **返回键逻辑**: 在桌面主页按下返回键时，**不退出应用**，而是弹出提示：“已经是桌面了”。
- **Home键逻辑**: 应用需配置为系统默认 Launcher，按下 Home 键始终返回本桌面。

## 3. 开发计划 (第一阶段)
1. **基础框架**: 配置 AndroidManifest，声明 Launcher 权限。
2. **主页布局**: 实现单页 Grid 布局，放置预设图标和“+”号。
3. **列表逻辑**: 读取本机应用列表，实现勾选添加功能。
4. **交互完善**: 实现返回键拦截提示。
