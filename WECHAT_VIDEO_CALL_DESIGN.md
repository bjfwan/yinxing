# 微信视频通话自动化技术方案

## 1. 项目背景

### 1.1 目标用户
- 视障老年人群体
- 手机操作能力有限
- 手机性能可能较差，响应速度慢

### 1.2 核心需求
- 一键发起微信视频通话
- 自动接听来电（微信视频、系统电话）
- 高稳定性、强容错性
- 实时反馈操作状态

### 1.3 技术约束
- 仅供私人使用，不上架应用商店
- 用户已授予所有必要权限
- 微信版本相对固定
- 需要适配低性能手机

## 2. 技术架构

### 2.1 核心技术栈
- AccessibilityService：界面自动化操作
- NotificationListenerService：监听来电通知
- TTS（TextToSpeech）：语音反馈
- SharedPreferences：存储配置和学习数据
- Handler + Coroutines：异步任务管理

## 3. 发起视频通话方案

### 3.1 整体流程

```
用户点击联系人
    ↓
检查前置条件（网络、权限、微信安装）
    ↓
启动微信应用
    ↓
等待微信首页加载完成（自适应等待）
    ↓
点击搜索图标/搜索框
    ↓
输入联系人名称
    ↓
等待搜索结果加载
    ↓
点击目标联系人
    ↓
等待聊天界面加载
    ↓
点击视频通话按钮
    ↓
完成
```

### 3.2 自适应超时机制

#### 3.2.1 初始超时配置
```kotlin
object TimeoutConfig {
    const val LAUNCH_WECHAT = 15000L
    const val LOAD_HOME = 20000L
    const val SEARCH_CONTACT = 15000L
    const val LOAD_CHAT = 15000L
```

#### 3.2.2 自学习机制
```kotlin
data class PerformanceMetrics(
    val launchWeChatTimes: MutableList<Long> = mutableListOf(),
    val loadHomeTimes: MutableList<Long> = mutableListOf(),
    val searchContactTimes: MutableList<Long> = mutableListOf(),
    val loadChatTimes: MutableList<Long> = mutableListOf()
) {
    fun getAdaptiveTimeout(step: String, defaultTimeout: Long): Long {
        val times = when(step) {
            "launch" -> launchWeChatTimes
            "home" -> loadHomeTimes
            "search" -> searchContactTimes
            "chat" -> loadChatTimes
            else -> return defaultTimeout
        }
        
        if (times.isEmpty()) return defaultTimeout
        
        val maxTime = times.maxOrNull() ?: defaultTimeout
        return maxOf(maxTime * 1.5, defaultTimeout).toLong()
    }
    
    fun recordTime(step: String, time: Long) {
        val times = when(step) {
            "launch" -> launchWeChatTimes
            "home" -> loadHomeTimes
            "search" -> searchContactTimes
            "chat" -> loadChatTimes
            else -> return
        }
        
        times.add(time)
        if (times.size > 10) {
            times.removeAt(0)
        }
    }
}
```

### 3.3 状态检测策略

#### 3.3.1 多重验证机制
每个状态使用多个特征进行验证，满足任意2个即认为状态正确

**微信已启动检测：**
- 前台应用包名 = "com.tencent.mm"
- 存在微信特征控件（标题栏、底部导航）
- Activity 名称包含 "LauncherUI"

**首页加载完成检测：**
- 搜索图标可见
- 底部导航栏存在（4个Tab）
- 聊天列表可滚动

**搜索框出现检测：**
- 搜索输入框可见
- 取消按钮存在
- 键盘已弹出

**联系人搜索结果检测：**
- 结果列表存在
- 包含目标联系人名称的控件
- 可点击状态

**聊天界面加载检测：**
- 顶部显示联系人名称
- 视频通话按钮存在
- 消息输入框可见

#### 3.3.2 轮询检测实现
```kotlin
suspend fun waitForState(
    stateName: String,
    timeout: Long,
    checkInterval: Long = 300L,
    validator: () -> Boolean
): Boolean = withContext(Dispatchers.IO) {
    val startTime = System.currentTimeMillis()
    
    while (System.currentTimeMillis() - startTime < timeout) {
        if (validator()) {
            return@withContext true
        }
        delay(checkInterval)
    }
    
    return@withContext false
}
```

### 3.4 渐进式操作策略

#### 3.4.1 搜索框未出现的处理
```
步骤1：等待10秒，检测搜索框
    ↓ 未出现
步骤2：尝试点击顶部搜索图标（坐标定位）
    ↓ 等待5秒
步骤3：尝试点击屏幕顶部中央区域（通用位置）
    ↓ 等待5秒
步骤4：模拟向下滑动（可能页面未在顶部）
    ↓ 等待5秒
步骤5：按返回键，重新进入微信
    ↓ 等待10秒
步骤6：最终失败，语音提示并返回桌面
```

#### 3.4.2 联系人未找到的处理
```
步骤1：等待10秒，检测搜索结果
    ↓ 未找到
步骤2：清空搜索框，重新输入（可能输入错误）
    ↓ 等待5秒
步骤3：尝试模糊搜索（只输入姓氏）
    ↓ 等待5秒
步骤4：检查联系人是否在最近聊天列表
    ↓ 等待5秒
步骤5：最终失败，语音提示"未找到联系人XXX"
```

## 4. 来电接听方案

### 4.1 微信视频来电

#### 4.1.1 来电检测
使用NotificationListenerService监听微信通知

**检测条件：**
- 通知来源：com.tencent.mm
- 通知标题包含："视频通话"或"语音通话"
- 通知内容包含联系人名称
- 通知有全屏Intent

#### 4.1.2 自动接听流程
```
检测到微信视频来电通知
    ↓
亮屏（如果息屏）
    ↓
解锁（如果无密码锁屏）
    ↓
显示自定义来电界面（覆盖在微信上）
    ↓
语音播报："XXX打来视频电话"
    ↓
等待用户操作或自动接听（可配置）
    ↓
使用AccessibilityService点击"接听"按钮
    ↓
隐藏自定义界面
```

#### 4.1.3 自定义来电界面设计

**布局元素：**
- 联系人头像（大尺寸，200dp）
- 联系人姓名（超大字体，48sp）
- 来电类型提示："视频通话"（32sp）
- 接听按钮（绿色，圆形，120dp）
- 挂断按钮（红色，圆形，120dp）

**交互设计：**
- 接听按钮：点击后自动点击微信接听按钮
- 挂断按钮：点击后自动点击微信挂断按钮
- 全屏显示，覆盖微信界面
- 背景模糊效果

### 4.2 系统电话来电

#### 4.2.1 来电检测
使用TelephonyManager监听电话状态

**监听广播：**
- ACTION_PHONE_STATE_CHANGED
- 状态：RINGING

**获取来电信息：**
- 来电号码
- 从联系人数据库匹配姓名
- 读取联系人头像

#### 4.2.2 自定义来电界面

**功能特性：**
- 显示大头贴照片（全屏背景）
- 超大字体显示姓名
- 语音播报："XXX打来电话"
- 超大接听/挂断按钮
- 支持快速回复短信（预设模板）

**接听操作：**
- 使用系统API接听电话
- 稳定性：99%
